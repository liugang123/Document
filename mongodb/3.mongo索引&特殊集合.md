- #### 索引类型
```
1.复合索引：建立在多个字段上的索引
2.覆盖索引：当一索引包含用户请求的所有字段，可以认为这个索引覆盖了本次查询
3.隐式索引：复合索引具有双重功能，而且对不同的查询可以表现为不同的索引
  复合索引{"age":1,"username":1}可以当作{"age":1}索引一样使用
4.唯一索引：可以确保集合中的每个文档的指定键都有唯一的值
  mongoDB会在创建集合时自动创建一个键"_id"的唯一索引
5.复合唯一索引：单个键的值可以相同但所有键的组合值必须是唯一的
6.稀疏索引：一个可能存在也可能不存在的字段，当它存在时，值必须是唯一的，这是可以使用unique和sparse选项组合使用创建稀疏索引
  MongoDB中的稀疏索引和关系数据库的稀疏索引是完全不同的概念
  >db.ensureIndex({"email":1},{"unique":true,"sparse":true})
```

- #### 索引管理
```
1.使用ensureIndex和dropIndex方法创建和删除索引
2.所有的数据库索引信息都存储在system.indexes集合中，这个保留集合，不能在其中插入或删除文档
3.创建索引后就可以在system.indexes中看到它的元信息
  通过db.collectionName.getIndexes()可以查看给定集合上所有索引信息
4.索引名称的默认形式是keyName1_dir1_keyName2_dir2_..._keyNameN_dirN
 keyNameX是索引的键，dirX是索引的方向(1或-1)
 也可以在ensureIndex中指定索引的名称
 >db.foo.ensureIndex({"a":1,"b":1,..,"z":1},...{"name":"alphabet"})
```

- #### 固定集合
```
1.普通集合：动态创建的，可以自动增长以容纳更多的数据
2.固定集合：预先创建好，而且大小是固定的，类似于循环队列
  当集合被占满时，如果再插入新文档，固定集合会自动将最老的文档从集合中删除
3.为固定集合指定文档数量限制时，必须同时指定固定集合的大小，不管先达到那个限制，之后插入的新文档就会把最老的文档挤出集合：
  固定集合的文档数量不能超过文档数量限制，固定集合的大小也不能超过大小限制
  >db.createCollection("my_collection",{"capped":true,"size":10000,"max":100})
4.自然排序：固定集合中的文档是按照文档被插入的顺序保存的，自然顺序就是文档的插入顺序
5.循环游标：当循环游标的结果集被取光后，游标不会被关闭，当有新文档插入到集合中时，循环游标会继续取到结果x
  循环游标类似tail -f命令，如果超过10分钟没有新的结果，循环游标就会被释放
```
