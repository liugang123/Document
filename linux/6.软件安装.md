- #### 原始码与Tarball
```
1.原始码其实大多是纯文本文档，需要通过编译程序的编译动作后，才能够制作出Linux系统能够认识的可执行的binary file
2.开放原始码可以加速软件的更新速度，让软件效能更快、漏洞修补更实时
3.在Linux系统中，最标准的C语言编译程序为gcc
4.在编译的过程中，可以藉由其他软件提供的函式库来使用该软件的相关机制与功能
5.为了简化编译过程中的复杂的指令输入，可以藉由make与makefile规则定义，
  来简化程序的更新、编译与连结等动作
  * make是一支程序，在执行make指令时，make会在当前目录下搜寻Makefile文本文件，makefile文件记录了原始码如何编译的详细信息
  * makefile的规则：
    目标(target)：目标文件1  目标文件2
    <tab>  gcc  -o 要建立的执行文件 目标文件1 目标文件2
6.Tarball为使用tar与gzip、bzip2、xz压缩功能所打包与压缩的，具有原始码的文件
7.要使用Tarball管理Linux系统上的软件，最好需要gcc，make，autoconfig、kernel source、kernel header等前驱软件才行，
  所以在安装Linux前，最好能够选择Software development以及kernel development之类的群组
8.函式库有动态函式库和静态函式库，动态函式库在升级上具有较佳的优势，
  动态函式库的扩展名为*.so而静态则为*.a
9.patch的主要功能在更新原始码，所以更新原始码之后，还需要进行重新编译的动作才行
10.可以使用ldconfig与/etc/ld.so.conf，/etc/ld.so.conf.d/*conf来制作动态函式库的链接与快取
11.通过MD5、SHA1、SHA256的编码可以判断下载的文件是否为原本厂商所释放出的文件
```

- #### RPM、SRPM和YUN
```
1.为了避免使用者自行编译的困扰，开发商自行在特定的硬件与操作系统平台上面预先编译好软件，并将软件以特殊格式封装成文件，
  提供终端用户直接安装到固定的操作系统上，并提供简单的查询、安装、移除等流程，
  此称为软件管理员，常见的软件管理员有RPM与DPKG两大主流
2.RPM的全名是RedHat Package Manager，原本是由Red Hat公司所发展的，流传甚广
3.RPM类型的软件中，所含有的软件是经过编译后的binary program，所以可以直接安装在用户端的系统上，
  不过，也因为此，所以RPM对于安装者的环境要求相当严格
4.RPM除了将软件安装到用户的系统上之外，还会将软件的版本、名称、文件与目录配置、系统需求等等，
  均记录在数据库(/var/lib/rpm)当中，方便未来的查询与升级、移除
5.RPM可针对不同的硬件等级来加以编译，制作出来的文件可于扩展名(i386,i586,i686)来分辨
6.RPM最大的问题为软件之间的相依性问题
  RPM的优点：
  * RPM内含已经编译过的程序与配置文件等数据，可以让用户免除重新编译的困扰 
  * RPM在被安装之前，会先检查系统的硬盘容量、操作心痛版本等，可避免文件被错误安装
  * PRM文件本身提供软件版本信息、相依属性软件名称、软件用途说明、软件所含文件等信息，便于了解软件
  * RPM管理的方式使用数据库记录RPM文件的相关参数，便于升级、移除、查询与验证
7.RPM安装文件 rpm -ivh package_name
  -i：install安装
  -v：观察更细节的安装信息画面
  -h：以安装信息列显示安装进度
  RPM升级更新
  -Uvh：若软件没有安装过，则系统进行安装，若软件有安装过旧版本，则系统自动更新至新版本
  -Fvh：若软件没有安装过，则系统不会安装，若软件安装过，则会被升级到最新版本
7.SRPM为source RPM，内含的文件为Source code而非binary file，所以安装时还需要经过compile；
  不过，SRPM最大的优点就是可以让使用者自行修改设定参数(makefile/configure的参数)，以符合使用者自己的Linux环境
8.RPM软件的属性相依问题，已经可以藉由yum或者APT等方式加以克服，CentOS使用的就是yum机制
9.yum服务器提供多个不同的软件库放置个别的软件，以提供客户端分别管理软件类别
```
